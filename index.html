How it works
The compiler pipeline
01 —
You write intentions
Describe what your application should do in natural language. No schemas, no boilerplate. Any level of precision.
02 —
SentenceState compiles
The runtime parses your intent into a canonical event log — a sequence of typed, timestamped, ordered state transitions.
03 —
The log is the app
The event log is the source of truth. Replay it to reproduce any state. Fork it. Audit it. Diff it across versions.
04 —
Deterministic output
Same log → same app. Every time. On every machine. The runtime is a pure function of your event log.
Platform features
Built for production
📼
Event log as source of truth
Your entire application is an append-only log of typed events. No hidden state. No side effects. Every transition is recorded, replayable, and forkable — from development to production.
🔬
Intent compiler
Natural language is parsed into structured intent trees, resolved against your application's type schema, and emitted as canonical events. Ambiguity is surfaced as a compile-time warning, not a runtime bug.
⏪
Time travel & replay
Scrub through your application's entire history. Reproduce any state exactly. Debug by replaying the log up to the failing event. Ship hotfixes by inserting corrective events — no migrations needed.
🔒
Deterministic runtime
The SentenceState runtime is a pure function: log in, application out. No external dependencies, no environment-specific behaviour. The same event log produces identical output in CI, staging, and production.
🌿
Log branching & merging
Branch your event log like a git repository. Experiment with alternative application histories. Merge branches with conflict resolution at the event level, not at the state level.
